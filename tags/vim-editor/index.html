Avoiding Mappings

normal命令会根据用户的key mappings执行相应的命令,也是说如果我们使用normal执行某些默认命令时(而此默认命令被映射成了别的命令),得到的可能不是想要的效果.我们需要一个能执行默认命令默认行为的normal命令,类似于nnoremap.

===分割线===

答案就是normal!

在编写VIM script时,我们应该始终使用normal!命令,不要相信用户~/.vimrc文件中的key mappings.

Chapter 30 Execute Normal!

Execute Normal! http://learnvimscriptthehardway.stevelosh.com/chapters/30.html

执行下面的命令

  1. 跳转至首行;
  2. 搜索foo;
  3. 删除搜索结果所在行

    在前面介绍normal!命令时, 执行:normal! /foo<cr>命令并没有我们想要的所有foo的效果,<cr>被解读成了4个单独的字符,而不是return. 现在联合使用execute和normal!,则该问题迎刃而解.

尝试下面的命令

  1. :execute "normal! ...": 在normal模式下执行...命令
  2. mq: 在当前位置设置mark“q”
  3. A: 将光标移至当前行尾,并进入insert入模式
  4. ;: 输入分号“;”
  5. \<esc>: 退出insert模式进入normal模式
  6. `q: 回到mark“q”所在位置
